Step-by-Step Deployment

Step 1
Log into cPanel and open File Manager

Go to your hosting provider's cPanel URL (usually yourdomain.co.za:2083 or via your host's client area).

Click File Manager → navigate to public_html. This is your web root — everything here is publicly accessible.

⚠ If public_html already has files (e.g. a WordPress site), do not delete them — scroll to Step 1b for subdomain deployment instead.
Step 2
Upload the landing page to public_html root

In File Manager, make sure you are inside public_html/.

Click Upload → upload the file: landing/index.html

This file should be named index.html at the root — it becomes your homepage at www.praeto.co.za.

Also upload the root .htaccess file. Important: in File Manager, hidden files (starting with .) may not show by default. Click Settings → tick Show Hidden Files.

Step 3
Create the /app folder and upload the React bundle

Still inside public_html/: click New Folder → name it app.

Navigate into public_html/app/.

Upload all files from your extracted app/ folder:

app/index.html
app/.htaccess
app/favicon.svg
app/icons.svg
app/assets/index-[hash].js

The easiest way: select all files in the app/ folder on your computer, then drag-and-drop into File Manager, or use Upload to select multiple files. Upload the assets/ subfolder separately by creating it first.

Step 4
Create the assets subfolder and upload the JS bundle

Inside public_html/app/: click New Folder → name it assets.

Navigate into public_html/app/assets/.

Upload the file: app/assets/index-C9lBq05w.js (the filename hash may differ slightly).

ℹ The JS bundle is ~300KB — it contains the entire member portal and admin panel compiled into one file. Upload may take 15-30 seconds on slower connections.
Step 5
Verify file permissions

All uploaded files should have permissions 644 (files) and 755 (folders).

In File Manager, right-click any file → Change Permissions → set to 644.

cPanel usually sets these correctly on upload, but double-check if you get 403 errors.

Step 6
Enable mod_rewrite (if not already active)

The .htaccess files require Apache's mod_rewrite module. This is enabled by default on virtually all cPanel hosts.

If you get errors after deployment, contact your host and ask them to confirm mod_rewrite and AllowOverride All are enabled for your account.

Step 7
Test your deployment

Open a browser and test these three URLs:

URLWhat You Should See
https://www.praeto.co.zaMarketing landing page with sanction counter and ticker
https://www.praeto.co.za/app/#/portalMember portal dashboard (logged in as Berkeley / Elite)
https://www.praeto.co.za/app/#/adminAdmin command centre — members, revenue, alerts, AI
✓ If all three load correctly, your platform is live. The full deployment takes under 10 minutes.

Troubleshooting

Landing page shows but /app gives a 404

The app/.htaccess file is missing or not uploaded. Go to public_html/app/ in File Manager, enable hidden files, and confirm .htaccess is present.

Blank white screen at /app

The JS bundle in assets/ is missing. Check that index-[hash].js exists inside public_html/app/assets/. Also check the browser console (F12 → Console) for the exact error.

HTTPS redirect not working

Go to cPanel → SSL/TLS → confirm a certificate is installed. Then go to AutoSSL and run it if needed. The .htaccess HTTPS redirect only works once SSL is active.

AI Advisor / AI Broadcast not responding

The AI features call the Anthropic API directly from the browser. Confirm you have active API access at console.anthropic.com. The API key is handled server-side by Anthropic's infrastructure — no key configuration is needed on your hosting.
