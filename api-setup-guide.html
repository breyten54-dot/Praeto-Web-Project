API Account Setup Guide
for Breyten

Three accounts to create — SendGrid, Anthropic, and PayFast.
Follow each section in order and note down every key before closing the tab.

Before you start: Keep a blank notepad or text file open. Each service gives you a key only once — if you close the tab without copying it, you have to regenerate it. At the end of this guide, paste all three keys into the .env file on the server. You don't need a credit card for Anthropic to start, and PayFast is free to register — you only pay per transaction.

Service 1 of 3 · Transactional Email
SendGrid
Sends welcome emails, payment receipts, password resets, and compliance alert broadcasts to members.
signup.sendgrid.com
1
Create a free Twilio SendGrid account
Go to signup.sendgrid.com. Fill in your name, email address, and a password. Use berkeley@praeto.co.za or Breyten's own email — either works. Click Create Account.

You'll get a verification email. Open it and click the confirmation link. This takes you back to SendGrid's dashboard.
2
Complete the short setup survey
SendGrid asks a few questions about how you'll use email. Answer like this:

• Sending purpose: Transactional
• Volume: < 10,000 emails per month
• Coding from scratch: Yes

Click through until you land on the main dashboard.
3
Verify your sender email address
In the left sidebar, go to Settings → Sender Authentication. Click Verify a Single Sender. Fill in:

• From Name: Praeto Compliance Club
• From Email: noreply@praeto.co.za
• Reply To: members@praeto.co.za

Click Create. SendGrid sends a verification email to noreply@praeto.co.za. Open that email (from the cPanel email account) and click the verification link.
NOTE

If noreply@praeto.co.za doesn't exist yet, create it first in cPanel → Email Accounts, then come back and verify. The email address must exist to receive the verification link.

4
Create your API key
In the left sidebar, go to Settings → API Keys. Click Create API Key (top right).

• Name: praeto-compliance-club
• Access: Full Access

Click Create & View. You'll see a long key starting with SG.
COPY NOW

Copy the entire key immediately and paste it into your notepad. SendGrid shows it only once — if you navigate away, you cannot see it again and will need to delete and regenerate.

# Paste this into your .env file
SENDGRID_API_KEY=SG.xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Service 2 of 3 · AI Compliance Advisor
Anthropic (Claude API)
Powers the AI Compliance Advisor — members ask compliance questions and get expert answers about FAIS, FIC Act, FSCA, and more.
console.anthropic.com
1
Create an Anthropic account
Go to console.anthropic.com. Click Sign Up. Enter your email address and create a password.

You'll receive a verification email — click the link in it to confirm your account. Once confirmed, you'll be taken to the Anthropic Console dashboard.
NOTE

You'll need to add a credit card before the API will work. Anthropic charges per use — Berkeley's typical usage for the AI Advisor will cost roughly R10–R40 per month depending on how many members use it. There's no monthly subscription fee on Anthropic's side.

2
Add a billing method
In the top navigation, click Settings → Billing. Click Add payment method and enter a credit or debit card. You don't need to buy credits upfront — Anthropic bills at the end of each month for what you used.

If you want a spending cap, click Set a usage limit and set it to something like $20/month to start.
3
Generate an API key
In the left sidebar, go to API Keys. Click Create Key.

• Name: praeto-compliance-club

Click Create Key. A key starting with sk-ant- appears.
COPY NOW

Copy the entire key and paste it into your notepad. Like SendGrid, this is shown only once. If you miss it, click Delete and create a new one.

# Paste this into your .env file
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
Service 3 of 3 · Payment Gateway
PayFast
Processes monthly membership subscriptions — Foundation (R2 500), Practitioner (R5 000), and Elite (R15 000). Money goes directly into Berkeley's bank account.
payfast.co.za
1
Register a PayFast merchant account
Go to payfast.co.za/registration and click Register as a Merchant.

Fill in Berkeley's personal details (not Breyten's) — PayFast requires the account holder to be the business owner because this is where payments are deposited:

• Full legal name: Berkeley Eldred Pretorius
• ID number: Berkeley's SA ID
• Email: berkeley@praeto.co.za
• Cell number: Berkeley's mobile
• Business name: Praeto Group Holdings / Praeto Trading CC
• Industry: Professional Services
IMPORTANT

PayFast will verify Berkeley's identity with FICA documents. He'll need to upload a certified copy of his SA ID and a bank statement or utility bill as proof of address (not older than 3 months). This is a legal FICA requirement — it's the same process as opening a bank account.

2
Link Berkeley's bank account
During registration, PayFast asks for a bank account to deposit payments into. Enter Berkeley's business bank account details:

• Bank name
• Account holder name
• Account number
• Branch code

PayFast does a small test deposit (a few cents) to verify the account. Berkeley will need to confirm the amount once it arrives.
3
Test in Sandbox mode first
While PayFast is completing the verification (usually 1–3 business days), we can test payments using PayFast's Sandbox. The system is already set up for this — the .env file has PAYFAST_ENV=sandbox by default.

PayFast provides test credentials for sandbox. Log into your PayFast account, go to Integrations → Sandbox and you'll see the test Merchant ID and Key. Use these for testing.
4
Get your live Merchant credentials
Once PayFast approves the account, log into payfast.co.za → Settings → Merchant Settings. You'll find three values:

• Merchant ID — a 8-digit number
• Merchant Key — a 16-character alphanumeric string
• Passphrase — set this yourself (Settings → Merchant Settings → Set Passphrase). Choose something like Praeto@PayFast2026 and note it down.
# Paste these into your .env file
PAYFAST_MERCHANT_ID=12345678
PAYFAST_MERCHANT_KEY=abc123def456ghi7
PAYFAST_PASSPHRASE=Praeto@PayFast2026
# Change from sandbox to production when account is approved:
PAYFAST_ENV=production
Once you have all three — here's where they go
Open the .env file in the backend folder on the server
backend/.env
SendGrid — paste your SG.xxx… key
SENDGRID_API_KEY=
Anthropic — paste your sk-ant-xxx… key
ANTHROPIC_API_KEY=
PayFast — Merchant ID, Key, and Passphrase
PAYFAST_MERCHANT_ID=
PAYFAST_MERCHANT_KEY=
PAYFAST_PASSPHRASE=
Once all keys are in, restart the backend server and test a payment in Sandbox
Then go live ✓
